# 项目手册：基于深度学习(MLP)与LLM Agent的自动化日志异常检测系统


## 1. 项目概述

本项目旨在利用**机器学习（深度学习分类）与大语言模型（Agent）**技术，解决工业界大规模系统日志监控的痛点。

- **核心逻辑**：通过神经网络（MLP）实时识别日志流中的异常模式，并调用Agent结合知识库进行自动化故障诊断。
- **硬核点**：涉及非结构化文本处理（NLP）、特征工程、不平衡数据集分类、以及Agent逻辑编排。

## 2. 核心技术栈

- **编程语言**：Python 3.9+
- **深度学习**：PyTorch（用于构建 MLP 分类器）
- **特征提取**：Regex / Drain 算法 (日志模板化)
- **智能体框架**：LangChain / LangGraph (Agent流程控制)
- **推理依据**：Ollama (本地运行 Llama 3) 或 DeepSeek API

## 3. 详细实施步骤

### 第一阶段：数据获取与清理（Data Collection）

- **下载数据集**：从loghub下载经典的HDFS 日志数据集（约1100万行）。
- **日志模板化**：
  > AI指令："请写一个Python脚本，使用正则表达式将HDFS日志中的IP地址、日期和动态ID替换为<*>，并提取出唯一的日志模板（Templates）。"
  
  目标：将杂乱的文字变成构造的"模板ID"。

### 第二阶段：特征工程（Feature Engineering）——关键硬核步骤

- **滑动窗口统计**：
  > AI指令："请帮我统计每1分钟（或固定行数）内，各个日志模板ID出现的频次。输出一个矩阵，行是时间窗口，列是模板ID，值为频次。"
  
- **数据标注**：根据HDFS数据集自带的anomaly_label文件，给每个窗口打上标签（0为正常，1为异常）。
- **归一化**：利用StandardScaler对频次数据进行缩放，加速模型收敛。

### 第三阶段：MLP分类模型训练（Model Training）——实验3核心

- **架构设计**：
  - 输入层：维度相同模板的总数。
  - 隐藏层：两层全连接层（例如256和64单元），使用ReLU激活函数。
  - 输出层：1个神经元，使用Sigmoid激活函数。
  
- **训练脚本**：
  > AI指令："请使用PyTorch写一个二分类MLP模型。针对数据不平衡的特点（异常极少），请在损失函数中使用BCEWithLogitsLoss并在训练时加入权重平衡。"
  
- **结果验证**：当前损失曲线和F1-Score（对于异常检测，F1比准确率更重要）。

### 第四阶段：Agent异常自愈闭环（Agent整合）

- **故障诊断逻辑**：当模型概率预测 $> 0.8$ 时，触发Agent。
- **LLM 提示词工程**：
  > AI指令："请帮我写一个LangChain提示词模板。输入是：'异常日志片段'和'历史维护手册'。要求LLM以SRE专家的身份分析故障原因，并给出三个排查建议。"
  
  大致上：编写简单的脚本，模拟发送邮件或钉钉。
