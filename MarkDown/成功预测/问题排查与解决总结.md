# HDFS 异常检测模型训练问题排查与解决总结

## 一、问题背景

在构建基于 Kafka + ClickHouse 的实时 HDFS 日志异常检测系统时，遇到了模型预测结果与官方结果不一致的问题。

### 原始数据流程

```
Kafka Producer (日志过滤) 
    → ClickHouse (物化视图统计)
    → 导出 CSV (block_features.csv)
    → combine.py (合并标签)
    → train_conbine.py (训练模型)
    → predict.py (预测)
```

---

## 二、问题表现

运行 `predict.py` 后，预测结果与官方结果存在明显差异：
- 部分 BlockId 不在官方统计中
- 部分 BlockId 的 Label 与官方相反

---

## 四、根本原因

### ❌ 错误的导出方式

```sql
-- 错误：导出的是长格式（block_id, event_id, cnt）
SELECT block_id, event_id, sum(cnt) AS total_cnt 
FROM block_event_stats 
GROUP BY block_id, event_id 
INTO OUTFILE 'Event.csv' 
FORMAT CSV;
```

这种导出方式是**长格式**，每个 block_id 会有多行记录（每个事件一行），无法直接用于模型训练。

### ❌ 缺少表头

ClickHouse 默认 `FORMAT CSV` 不包含表头，需要使用 `FORMAT CSVWithNames`。

---

## 五、解决方案

### ✅ 正确的导出方式

```sql
SELECT 
    block_id AS BlockId,
    sumIf(cnt, event_id = 'E1') AS E1,
    sumIf(cnt, event_id = 'E2') AS E2,
    sumIf(cnt, event_id = 'E3') AS E3,
    sumIf(cnt, event_id = 'E4') AS E4,
    sumIf(cnt, event_id = 'E5') AS E5,
    sumIf(cnt, event_id = 'E6') AS E6,
    sumIf(cnt, event_id = 'E7') AS E7,
    sumIf(cnt, event_id = 'E8') AS E8,
    sumIf(cnt, event_id = 'E9') AS E9,
    sumIf(cnt, event_id = 'E10') AS E10,
    sumIf(cnt, event_id = 'E11') AS E11,
    sumIf(cnt, event_id = 'E12') AS E12,
    sumIf(cnt, event_id = 'E13') AS E13,
    sumIf(cnt, event_id = 'E14') AS E14,
    sumIf(cnt, event_id = 'E15') AS E15,
    sumIf(cnt, event_id = 'E16') AS E16,
    sumIf(cnt, event_id = 'E17') AS E17,
    sumIf(cnt, event_id = 'E18') AS E18,
    sumIf(cnt, event_id = 'E19') AS E19,
    sumIf(cnt, event_id = 'E20') AS E20,
    sumIf(cnt, event_id = 'E21') AS E21,
    sumIf(cnt, event_id = 'E22') AS E22,
    sumIf(cnt, event_id = 'E23') AS E23,
    sumIf(cnt, event_id = 'E24') AS E24,
    sumIf(cnt, event_id = 'E25') AS E25,
    sumIf(cnt, event_id = 'E26') AS E26,
    sumIf(cnt, event_id = 'E27') AS E27,
    sumIf(cnt, event_id = 'E28') AS E28,
    sumIf(cnt, event_id = 'E29') AS E29
FROM block_event_stats
GROUP BY block_id
INTO OUTFILE 'block_features.csv'
FORMAT CSVWithNames;
```

### ✅ combine.py 调整列顺序

```python
import pandas as pd

features = pd.read_csv('block_features.csv')
labels = pd.read_csv('Event.csv')

df = features.merge(labels, on='BlockId', how='inner')
df['target'] = df['Label'].map({'Success': 0, 'Fail': 1})

# 调整列顺序：BlockId, Label, target, E1~E29
feature_cols = [f'E{i}' for i in range(1, 30)]
new_order = ['BlockId', 'Label', 'target'] + feature_cols
df = df[new_order]

df.to_csv('training_data.csv', index=False)
```

## 六、关键要点总结

| 问题点 | 错误做法 | 正确做法 |
|--------|----------|----------|
| 导出格式 | 长格式 (block_id, event_id, cnt) | 宽格式 (block_id, E1, E2, ..., E29) |
| 表头 | `FORMAT CSV` 无表头 | `FORMAT CSVWithNames` 带表头 |
| 数据类型 | sum(cnt) | sumIf(cnt, event_id = 'EX') |
| 列合并 | 任意顺序 | BlockId → Label → target → E1~E29 |
