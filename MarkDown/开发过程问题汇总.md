## 问题一：conda虚拟环境创建位置问题

### 问题描述
用户希望在项目目录（`E:\private_project\AI_application`）中创建conda虚拟环境，而不是将依赖安装到conda的默认目录（`D:\anaconda\envs`）。

### 解决方案
使用conda的`--prefix`参数指定虚拟环境的创建位置。

### 代码示例
```bash
# 在当前目录创建conda虚拟环境
D:\anaconda\Scripts\conda.exe create --prefix E:\private_project\AI_application\ai_application_conda python=3.11

# 激活环境
D:\anaconda\Scripts\conda.exe activate E:\private_project\AI_application\ai_application_conda
```

### 关键参数说明
| 参数 | 说明 |
|------|------|
| `--prefix` | 指定虚拟环境的安装路径 |
| `python=3.11` | 指定Python版本（推荐3.10或3.11，适合RAG+Agent项目） |

## 问题二：删除重复的虚拟环境

### 问题描述
项目中存在两个虚拟环境文件夹：
- `ai_application/` - Python venv创建的
- `ai_application_conda/` - conda创建的

### 解决方案
删除venv创建的虚拟环境，保留功能更强大的conda环境。

### 代码示例
```powershell
# 使用PowerShell删除venv环境
Remove-Item -Recurse -Force E:\private_project\AI_application\ai_application
```

### 注意事项
- 删除前确认conda环境已正常工作
- 删除后需在PyCharm中重新配置Python解释器路径

## 问题四：dashscope模块未安装

### 问题描述
运行代码时出现错误：
```
ModuleNotFoundError: No module named 'dashscope'
```
用户在全局环境安装了dashscope，但代码运行在conda虚拟环境中。

### 解决方案
确保在正确的conda虚拟环境中安装依赖包。

### 代码示例
```bash
# 激活conda环境后安装
conda activate E:\private_project\AI_application\ai_application_conda
pip install dashscope

# 或者直接使用conda环境的pip
E:\private_project\AI_application\ai_application_conda\Scripts\pip.exe install dashscope
```

### 关键点
- **环境匹配**：安装的包必须与代码运行的环境一致
- **检查当前环境**：运行`python --version`确认当前Python路径

## 问题五：path_tool.py路径问题

### 问题描述
用户发现输出的路径混合了反斜杠和正斜杠：
```
E:\private_project\AI_application\HDFS_v1/HDFS.log
```

### 解决方案
这是正常现象，Python的`os.path.join()`会自动处理路径分隔符。在Windows上显示时可能会有混合，但不影响实际使用。

如果需要统一格式，可以使用`pathlib`：

### 代码示例
```python
from pathlib import Path

def get_abs_path(relative_path: str) -> str:
    project_root = get_project_root()
    abs_path = Path(project_root) / relative_path
    return str(abs_path)  # 或使用 abs_path.as_posix() 获取纯正斜杠
```

## 问题六：RAG检索返回"未找到相关信息"

### 问题描述
运行agent_tools.py测试时，检索结果始终返回"未找到相关信息"，LLM只能返回模板回答。

### 根本原因
向量数据库（Chroma）中没有任何文档数据。

### 解决方案
需要先将知识文档加载到向量数据库中。

#### 步骤1：创建知识文档目录
在项目根目录创建`hdfs_knowledge`文件夹，添加txt格式的知识文档：

```
E:\private_project\AI_application\
├── hdfs_knowledge\
│   ├── hdfs_intro.txt
│   ├── hdfs_read_write.txt
│   └── hdfs_troubleshooting.txt
```

#### 步骤2：运行导入脚本

创建`load_knowledge.py`：

```python
import sys
import os

sys.path.insert(0, os.path.dirname(os.path.abspath(__file__)))

from rag.vector_store import VectorStoreService

def load_knowledge():
    """加载知识库文档到向量数据库"""
    print("=" * 60)
    print("开始加载知识库文档...")
    print("=" * 60)
    
    vector_store = VectorStoreService()
    
    print("\n正在加载文档...")
    vector_store.load_document()
    
    print("\n" + "=" * 60)
    print("知识库加载完成！")
    print("=" * 60)

if __name__ == "__main__":
    load_knowledge()
```

运行导入：
```bash
E:\private_project\AI_application\ai_application_conda\python.exe E:\private_project\AI_application\load_knowledge.py
```

#### 步骤3：验证导入结果
导入成功后，重新运行agent_tools.py测试：
```bash
E:\private_project\AI_application\ai_application_conda\python.exe E:\private_project\AI_application\agent\tools\agent_tools.py
```

## 问题七：Agent Tools中@tool装饰器的使用

### 问题描述
用户需要创建供Agent调用的工具函数，希望使用LangChain的`@tool`装饰器。

### 解决方案
使用`langchain_core.tools`模块中的`@tool`装饰器定义工具函数。

### 代码示例
```python
from langchain_core.tools import tool
from rag.rag_service import RagSummarizerService

@tool(description="从向量存储中检索参考资料，用于回答用户问题")
def rag_summarize(query: str) -> str:
    """
    从RAG知识库中检索相关信息并生成回答
    
    Args:
        query: 用户的问题或查询内容
        
    Returns:
        str: 基于知识库生成的回答
    """
    rag = RagSummarizerService()
    return rag.summarize(query)

@tool(description="获取当前时间")
def get_current_time() -> str:
    from datetime import datetime
    return datetime.now().strftime("%Y-%m-%d %H:%M:%S")

# 测试工具
if __name__ == "__main__":
    result = rag_summarize.invoke("什么是HDFS日志？")
    print(result)
```

### 工具列表

| 工具函数 | 描述 |
|---------|------|
| `rag_summarize` | 从RAG知识库检索并生成回答 |
| `rag_retrieve` | 仅检索相关文档 |
| `get_current_time` | 获取当前时间 |
| `calculate` | 计算数学表达式 |
| `preprocess_hdfs_logs` | 预处理HDFS日志 |
| `train_mlp_model` | 训练MLP模型 |
| `detect_anomaly` | 检测异常 |

## 常见错误排查

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| `ModuleNotFoundError: No module named 'xxx'` | 包未在当前环境安装 | 切换到正确的conda环境后安装 |
| `未找到相关信息` | 向量数据库为空 | 运行load_knowledge.py导入文档 |
