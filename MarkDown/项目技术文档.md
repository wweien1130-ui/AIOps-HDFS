# HDFS智能异常检测系统 - 完整技术文档

## 📋 项目概述

## 🔧 核心技术概念

### 1. 特征工程 (Feature Engineering)

**定义**：将原始数据转换为机器学习模型能够处理的数值特征的过程。

**在本项目中的应用**：

```
原始日志:
blk_123: Receiving block src:/10.1.1.1 dest:/10.1.1.2

步骤1: 模板化
     ↓
Receiving block src:* dest:* → E5

步骤2: 事件计数
     ↓
| BlockId | E1 | E2 | E3 | E4 | E5 | ... |
|---------|----|----|----|----|----|-----|
| blk_1   | 0  | 0  | 0  | 0  | 1  | ... |

步骤3: 归一化
     ↓
使用 StandardScaler 对特征进行标准化
```

### 2. MLP (多层感知机)

**定义**：一种前馈神经网络，由输入层、隐藏层和输出层组成。

**本项目中的网络结构**：

```python
class MLP(nn.Module):
    def __init__(self, input_dim):
        super(MLP, self).__init__()
        self.fc1 = nn.Linear(input_dim, 128)   # 输入层 → 隐藏层1
        self.relu1 = nn.ReLU()                  # 激活函数
        self.fc2 = nn.Linear(128, 32)          # 隐藏层1 → 隐藏层2
        self.relu2 = nn.ReLU()                  # 激活函数
        self.fc3 = nn.Linear(32, 1)            # 隐藏层2 → 输出层
    
    def forward(self, x):
        x = self.fc1(x)
        x = self.relu1(x)
        x = self.fc2(x)
        x = self.relu2(x)
        x = self.fc3(x)
        return x
```

**参数说明**：

| 层 | 输入维度 | 输出维度 | 参数数量 |
|----|----------|----------|----------|
| fc1 | 29 | 128 | 29×128 + 128 = 3840 |
| fc2 | 128 | 32 | 128×32 + 32 = 4128 |
| fc3 | 32 | 1 | 32×1 + 1 = 33 |
| **总计** | - | - | **8001** |

### 3. 早停机制 (Early Stopping)

**目的**：防止过拟合，在验证集性能不再提升时停止训练。

**原理**：

```
Epoch 1:  Loss = 0.5   → 保存模型
Epoch 2:  Loss = 0.3   → 保存模型 (最佳)
Epoch 3:  Loss = 0.25  → 保存模型
Epoch 4:  Loss = 0.28  → 不保存 (性能下降)
Epoch 5:  Loss = 0.30  → 不保存 (继续下降)
...
Epoch 10: Loss = 0.35  → 早停! 连续patience次没有改善
```

**代码实现**：

```python
best_loss = float('inf')
patience = 10      # 允许连续没有改善的次数
no_improve_count = 0
best_model_state = None

for epoch in range(epochs):
    # ... 训练代码 ...
    
    if avg_loss < best_loss:
        best_loss = avg_loss
        no_improve_count = 0
        best_model_state = model.state_dict().copy()  # 保存最佳模型
    else:
        no_improve_count += 1
    
    if no_improve_count >= patience:
        print(f'Early stopping at epoch {epoch + 1}')
        break

# 加载最佳模型
model.load_state_dict(best_model_state)
```

## 💡 常见问题排查

### 问题1：模型加载失败

**症状**：
```
RuntimeError: Error(s) in loading state_dict... size mismatch
```

**原因**：训练和推理时的模型架构不一致

**解决**：确保 MLP 架构为 128→32→1

---

### 问题2：GPU不可用

**症状**：
```
Using device: cpu
```

**原因**：安装的是CPU版本的PyTorch

**解决**：安装CUDA版本
```bash
pip install torch --index-url https://download.pytorch.org/whl/cu124
```

---

### 问题3：RAG返回模板而非答案

**症状**：返回的是prompt模板说明文档

**原因**：prompt文件配置错误或RAG chain配置问题

**解决**：检查 `prompts/rag_summarize.txt` 是否存在，或直接调用大模型

## 🔬 技术细节

### 正则表达式模板匹配

**问题**：原始模板使用 `[*]` 作为通配符，直接替换为 `<*>` 无法匹配

**解决方案**：

```python
# 将模板转换为正则表达式
def load_templates(self, template_file):
    self.templates = []
    for _, row in data.iterrows():
        event_id = row['EventId']
        template = row['EventTemplate']
        # 将 [*] 转换为正则表达式 .*
        pattern = template.replace('[*]', '.*')
        pattern = '^' + pattern + '$'
        self.templates.append((event_id, re.compile(pattern)))
```
