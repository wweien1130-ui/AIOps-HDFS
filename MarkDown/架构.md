## 📂 各模块详解

### 1️⃣ `api/` - API 服务层

| 文件 | 功能 |
|------|------|
| `main.py` | FastAPI 应用入口，定义所有 REST 接口（CORS、文件上传、Agent 对话、MLP 模型管理） |

**关键代码片段：**
```python
from fastapi import FastAPI, UploadFile, File
from agent.react_agent import ReactAgent

app = FastAPI(title="AI Application API", version="1.0.0")

# Agent 单例（保持对话记忆）
_agent_instance = None

def get_agent():
    global _agent_instance
    if _agent_instance is None:
        _agent_instance = ReactAgent()
    return _agent_instance
```

---

### 2️⃣ `agent/` - Agent 核心

| 文件 | 功能 |
|------|------|
| `react_agent.py` | ReAct (Reasoning + Acting) Agent 实现，调用 LLM 并协调各种工具 |
| **tools/** | Agent 可调用的工具集 |
| `agent_tools.py` | 工具函数：RAG检索、异常检测、离线批处理等 |
| `data_preparator.py` | HDFS 日志数据预处理（特征提取、标准化） |
| `realtime_query.py` | 实时异常查询（从 Kafka/Redis 获取数据） |

**核心关系：**
```
react_agent.py
    ├── 导入 model.factory.chat_models (通义千问)
    ├── 导入 utils.prompt_loader (加载系统提示词)
    ├── 创建 Agent 并注册工具集
    │   ├── rag_retrieve          → 从 Chroma 检索知识
    │   ├── detect_anomaly        → MLP 模型预测
    │   ├── preprocess_hdfs_logs  → 日志预处理
    │   └── realtime_query        → 实时异常查询
    └── execute_stream()          → 流式执行查询
```

---

### 3️⃣ `model/` - 机器学习层

| 文件 | 功能 |
|------|------|
| `factory.py` | 模型工厂，生产 ChatTongyi（通义千问）和 DashScope 嵌入模型 |
| `mlp_model.py` | MLP 神经网络实现，用于日志异常分类预测 |

**模型工厂代码：**
```python
from langchain_community.chat_models import ChatTongyi
from langchain_community.embeddings import DashScopeEmbeddings

class ChatModelFactory(BaseModelFactory):
    def generator(self):
        return ChatTongyi(model=rag_config["vector_store"]["chat_model"])

class EmbeddingsFactory(BaseModelFactory):
    def generator(self):
        return DashScopeEmbeddings(model=rag_config["vector_store"]["embedding"]["model"])

chat_models = ChatModelFactory().generator()
embedding_models = EmbeddingsFactory().generator()
```

---

### 4️⃣ `utils/` - 工具函数

| 文件 | 功能 |
|------|------|
| `config_handler.py` | 加载 YAML 配置文件（agent、base、llm、rag 等配置） |
| `path_tool.py` | 获取项目绝对路径的工具函数 |
| `logger_handler.py` | 统一日志管理 |
| `file_handler.py` | 文件读写操作 |
| `prompt_loader.py` | 加载系统提示词（system prompts） |

**配置加载示例：**
```python
import yaml

def load_rag_config(config_path: str = get_abs_path("config/rag.yml")):
    with open(config_path, "r", encoding="utf-8") as f:
        return yaml.load(f, Loader=yaml.FullLoader)

rag_config = load_rag_config()
```

---

### 5️⃣ `config/` - 配置文件

| 文件 | 功能 |
|------|------|
| `agent.yml` | Agent 的思考策略、超时设置等 |
| `llm.yml` | LLM 提供商配置（Ollama、DeepSeek、通义千问） |
| `rag.yml` | RAG 配置：向量存储、嵌入模型、聊天模型 |
| `chroma.yml` | Chroma 向量数据库配置 |
| `clickhouse.yaml` | ClickHouse 数据库连接配置 |
| `kafka.yml` | Kafka 消息队列配置 |
| `redis.yaml` | Redis 缓存配置 |

## 🔄 数据流程

```
用户提问
    ↓
Streamlit UI (app.py)
    ↓
FastAPI API (api/main.py)
    ↓
ReactAgent (agent/react_agent.py)
    ├── 1️⃣ 思考：调用通义千问模型分析问题
    ├── 2️⃣ 行动：根据思考结果选择工具
    │   ├── RAG检索 → 从 Chroma 向量库检索相关知识
    │   ├── MLP检测 → 使用 PyTorch 模型预测异常
    │   ├── 数据处理 → 预处理 HDFS 日志
    │   └── 实时查询 → 从 Kafka/Redis 获取实时数据
    └── 3️⃣ 观察：根据工具返回结果继续推理
    ↓
返回结构化响应
```

## 📊 技术栈总结

| 层级 | 技术 |
|------|------|
| 前端 | Streamlit |
| API | FastAPI + Uvicorn |
| Agent 框架 | LangChain |
| LLM | 阿里云通义千问 (ChatTongyi) |
| 嵌入模型 | DashScope Embeddings |
| 向量数据库 | Chroma |
| ML 模型 | PyTorch MLP |
| 配置管理 | YAML |
| 外部存储 | ClickHouse, Kafka, Redis |

---

这是一个典型的 **LLM Agent + RAG + ML** 混合架构，主要用于自动化日志异常检测和诊断。
